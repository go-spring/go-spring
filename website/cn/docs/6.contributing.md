## 如何参与

- 提交 Issue 反馈问题或提出需求建议
- 改进文档、示例或教程
- 修复 Bug 或补充测试
- 贡献新功能或基础设施集成

## 提交前准备

贡献代码前，请先阅读仓库根目录的 `CODING_STYLE.md`，了解项目编码规范。

建议在提交 PR 前完成必要的测试，并确保改动保持简单、清晰、直接。

## 记录踩坑

和 AI 协作、或日常开发中踩过的、值得别人绕开的坑，用 GitHub issue 记录——每个坑一条 issue，自带时间线、讨论和解决过程，天然可搜索、可链接。

- **怎么记**：开一条 issue，打上 `gotcha` 标签，标题写清现象，正文写清触发场景、原因和规避方式。
- **怎么查**：看标签筛选页，永远是最新的完整列表——[标签为 `gotcha` 的全部 issue](https://github.com/go-spring/go-spring/issues?q=label%3Agotcha)。
- **什么时候升级**：如果某个坑的规避方式变得“无需判断即可照做”，就把它提炼进根目录的 `CLAUDE.md`，issue 随之关闭。
