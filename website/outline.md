# 四、启停

1. 启动约束
    1. 启动流程线性、不可回滚
    2. init 阶段仅允许注册元数据，不允许执行逻辑
    3. 启动失败即进程退出，不清理已初始化资源
    4. runner 为一次性执行单元，不允许启动后台 goroutine
    5. 日志必须在进程退出前完成 flush
    6. server ready 之后发生异常进入 shutdown 流程
2. 启动主流程
    1. 配置失败时立即终止启动，不清理已初始化资源
    2. IoC 刷新成功后释放非必要资源
3. Runner
    1. 默认收集顺序不固定
    2. 可通过配置项指定顺序
    3. runner 间原则上不应存在依赖关系
4. Server
    1. start 过程中 panic 视为 error
    2. socket 优先完成 listen
    3. listen 失败时程序退出
5. 退出流程
    1. shutdown 幂等，可重复调用
    2. 可监听 Exiting 标记
6. 生命周期回调
    1. 当前不支持，未来如有必要再引入
7. 调试与日志
    1. 支持启停关键节点日志输出
