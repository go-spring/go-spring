# Bug 1：parseExpr 重复键漏检

- 状态：已提交
- 位置：`log_refresh.go`
- 预期来源：`parseExpr` 注释声明展开内联表达式时会检测重复键；配置解析应对同一最终键给出稳定错误，而不是依赖 Go map 遍历顺序。
- 问题：当输入同时包含 `db!` 和 `db.host` 这类会展开为同一最终键的配置时，如果普通键在内联表达式之后遍历，当前实现会直接覆盖已写入的值，不报重复键错误。
- 影响：同一份配置可能因 map 遍历顺序随机成功或失败；成功时用户的配置值被静默覆盖，导致日志配置行为不可预测。
- 复现：补充 `TestParseExprDuplicateExpandedKey`，重复调用 `parseExpr` 验证所有遍历顺序都必须返回重复键错误。
- 修复：普通键写入 `ret` 时也检查是否已存在；内联表达式分支继续检查展开后的子键。
- 验证：`go test ./...` 通过。
- 提交：本提交 `fix parseExpr duplicate key detection`
- 备注：只修复重复键检测，不调整内联表达式语法或其他刷新流程。
