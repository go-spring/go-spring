# starter-swagger

[English](README.md) | [中文](README_CN.md)

`starter-swagger` serves an interactive [Swagger UI](https://github.com/swagger-api/swagger-ui)
page and the OpenAPI document generated by `gs-http-gen --openapi`, so a
Go-Spring service can expose browsable API docs with no vendored assets and no
extra listening port.

## Installation

```bash
go get go-spring.org/starter-swagger
```

## Quick Start

### 1. Generate the OpenAPI Document

Produce `openapi.json` from your IDL with the HTTP code generator:

```bash
gs-http-gen --openapi -i api.thrift -o .
```

### 2. Import the `starter-swagger` Package

Refer to the [example.go](example/example.go) file.

```go
import _ "go-spring.org/starter-swagger"
```

### 3. Configure the UI

Add Swagger configuration in your project's [configuration file](example/conf/app.properties):

```properties
spring.swagger.specFile=openapi.json
spring.swagger.basePath=/swagger
spring.swagger.title=My API Docs
```

### 4. Reach the Docs

The starter contributes the UI as an `endpoint.Endpoint`. If you run
`starter-actuator`, the page is auto-mounted on the management port with no
wiring. Otherwise inject the `*StarterSwagger.UI` bean and mount it on your own
HTTP server:

```go
gs.Provide(func(ui *StarterSwagger.UI) *gs.HttpServeMux {
    mux := http.NewServeMux()
    mux.Handle(ui.Path(), ui)
    return &gs.HttpServeMux{Handler: mux}
})
```

Then open `http://<host>/swagger/` in a browser.

## Core Features

The [example.go](example/example.go) program demonstrates and asserts three things:

* **UI shell** — `GET /swagger/` returns an HTML page that boots Swagger UI.
* **Index alias** — `GET /swagger/index.html` serves the same shell.
* **Spec passthrough** — `GET /swagger/openapi.json` returns the generated OpenAPI document verbatim.

## Design Notes

* **No vendored assets**: the CSS/JS bundle is loaded from a CDN
  (`spring.swagger.assetBaseURL`, default `unpkg.com/swagger-ui-dist@5`) at
  runtime, so the starter ships only a small HTML shell. Point it at a
  self-hosted mirror for air-gapped environments.
* **Fail fast**: the OpenAPI document is read once at startup; a missing or
  unreadable spec surfaces immediately rather than 404ing after the app is live.
* **No port of its own**: the UI is contributed as a bean/`endpoint.Endpoint`;
  it is mounted on a server the application already runs (the actuator's
  management port or the app's own HTTP server).
* **Toggle without code changes**: set `spring.swagger.enabled=false` to disable
  the docs in production while keeping the import.
