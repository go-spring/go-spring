# netutil 设计说明
[English](DESIGN.md) | [中文](DESIGN_CN.md)

属于零依赖的 `stdlib` 层。当前只提供框架注册与启动日志用到的本机 IPv4
查询。

## 1. 职责与边界

- 提供一步到位的"我在网络中的地址是什么？"，供服务注册、日志打标、
  actuator info 使用。
- 不是网络框架。接口枚举、CIDR 匹配、地址解析等能力已由 `net` 覆盖，不
  重复造。

## 2. 设计说明

- 用 `sync.Once` 做缓存，进程周期内结果稳定，与框架"启动后网络地址固定"
  的默认假设一致。
- 出错时用 `"0.0.0.0"` 占位，保持 API 只返回字符串。代价是错配置会被静默
  吞掉，需要硬失败的调用方应直接使用 `net.InterfaceAddrs`。
- 仅支持 IPv4，因为 Go-Spring 目前面向的注册中心（Nacos / etcd / Consul
  风格）与日志系统主流仍以 IPv4 为键。
